# Types

> A glossary of types.

<SectionBreak />

## Account

| Name   | Type                       | Required | Default     | Description                                                                                                                          |
|--------|----------------------------|----------|-------------|--------------------------------------------------------------------------------------------------------------------------------------|
| __type | `connected` \| `ephemeral` | yes      | -           | The type of account: `connected` or `ephemeral`.                                                                                     |
| key    | `string`                   | yes      | -           | The public key of the account encoded with <a href={BASE_58_SPECIFICATION_URL} rel="noopener noreferrer" target="_blank">base58</a>. |
| name   | `string`                   | no       | `undefined` | A human-readable name for this account.                                                                                              |

<SectionBreak />

## BaseError

| Name             | Type     | Required | Default | Description                                  |
|------------------|----------|----------|---------|----------------------------------------------|
| message          | `string` | yes      | -       | A message associated with the error.         |
| isKatavaultError | `true`   | yes      | -       | Used to determine if this a Katavault error. |
| type             | `string` | yes      | -       | A reference for this error.                  |

<SectionBreak />

## CreateKatavaultParameters

| Name   | Type                                        | Required | Default | Description                                                                                                                                                                                      |
|--------|---------------------------------------------|----------|---------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| debug  | `boolean`                                   | no       | `false` | Whether to log debug messages.                                                                                                                                                                   |
| chains | `[ChainConstructor, ...ChainConstructor[]]` | yes      | -       | A collection of chains. It is **RECOMMENDED** to use <a href={KIBISIS_CHAINS_NPM_PACKAGE_URL} rel="noopener noreferrer" target="_blank">`@kibisis/chains`</a> to handle the chain configuration. |
| client | `{ icon?: string; name: string; }`          | no       | -       | The client's name and icon URL.                                                                                                                                                                  |

## SendRawTransactionParameters

| Name        | Type         | Required | Default | Description                                                                                               |
|-------------|--------------|----------|---------|-----------------------------------------------------------------------------------------------------------|
| chainID     | `string`     | yes      | -       | The <a href={CAIP_002_SPECIFICATION_URL} rel="noopener noreferrer" target="_blank">CAIP-002</a> chain ID. |

| Name        | Type         | Required | Default | Description                                                                                               |

<SectionBreak />

## SendRawTransactionResult

| Name          | Type                            | Required | Default | Description                                                                                                                                           |
|---------------|---------------------------------|----------|---------|-------------------------------------------------------------------------------------------------------------------------------------------------------|
| error         | [`BaseError`][errors] \| `null` | yes      | `null`  | If the transaction submission was unsuccessful, the exact error will be available.                                                                    |
| success       | `boolean`                       | yes      | -       | Whether the transaction submission was successful.                                                                                                    |
| transactionID | `string` \| `null`              | yes      | `null`  | If the transaction was sent successfully created, this will be the transaction ID. The structure and encoding will be different for different chains. |

| Name | Type     | Required | Default | Description                                                                                                                     |
|------|----------|----------|---------|---------------------------------------------------------------------------------------------------------------------------------|
| key  | `string` | yes      | -       | The <a href={BASE_58_SPECIFICATION_URL} rel="noopener noreferrer" target="_blank">base58</a> encoded public key of the account. |

<SectionBreak />

## SignMessageParameters

| Name       | Type                     | Required | Default     | Description                                                                                                                                                   |
|------------|--------------------------|----------|-------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------|
| accountKey | `string`                 | yes      | -           | The <a href={BASE_58_SPECIFICATION_URL} rel="noopener noreferrer" target="_blank">base58</a> encoded public key of the account that will be used for signing. |
| chainID    | `string`                 | yes      | -           | The <a href={CAIP_002_SPECIFICATION_URL} rel="noopener noreferrer" target="_blank">CAIP-002</a> chain ID.                                                     |
| encoding   | `base64` \| `hex`        | no       | `undefined` | The output encoding of the signature. If no encoding is specified, the signature will be in raw bytes (`Uint8Array`).                                         |
| message    | `string` \| `Uint8Array` | yes      | -           | A UTF-8 encoded message or raw bytes to sign.                                                                                                                 |

<SectionBreak />

## SignRawTransactionParameters

| Name        | Type         | Required | Default | Description                                                                                                                                                   |
|-------------|--------------|----------|---------|---------------------------------------------------------------------------------------------------------------------------------------------------------------|
| accountKey  | `string`     | yes      | -       | The <a href={BASE_58_SPECIFICATION_URL} rel="noopener noreferrer" target="_blank">base58</a> encoded public key of the account that will be used for signing. |
| chainID     | `string`     | yes      | -       | The <a href={CAIP_002_SPECIFICATION_URL} rel="noopener noreferrer" target="_blank">CAIP-002</a> chain ID.                                                     |
| transaction | `Uint8Array` | yes      | -       | The raw transaction data to be signed. This will be different for different chains.                                                                           |

<SectionBreak />

## SignRawTransactionResult

| Name      | Type                            | Required | Default | Description                                                                           |
|-----------|---------------------------------|----------|---------|---------------------------------------------------------------------------------------|
| error     | [`BaseError`][errors] \| `null` | yes      | `null`  | If the transaction signing was unsuccessful, the exact error will be available.       |
| success   | `boolean`                       | yes      | -       | Whether the transaction signing was successful.                                       |
| signature | `Uint8Array`                    | yes      | -       | The signature of the signed transaction data or null if the signing was unsuccessful. |

<SectionBreak />

## UserInformation

| Name        | Type     | Required | Default | Description                                                                                                   |
|-------------|----------|----------|---------|---------------------------------------------------------------------------------------------------------------|
| displayName | `string` | no       | -       | A human-readable display name for the user. This does not have to be unique and is used to display in the UI. |
| username    | `string` | yes      | -       | A globally unique identifier for the user. This could be, for example, an email address.                      |
